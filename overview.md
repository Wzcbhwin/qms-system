# QMS 质量整改追踪系统 — 项目概览

## 完成内容

构建了完整的「质量QMS整改追踪系统」，实现质量问题全生命周期闭环管理（发现→分析→整改→验证→归档），包含 5 大核心模块。

## 功能模块

### 模块1：问题录入中心（双入口）
- **批量导入**：CSV 模板下载、拖拽上传、自动校验（必填项/日期格式/字典匹配）、导入结果反馈（成功/失败明细）
- **单条在线添加**：4 步分步表单（Step Form），对应 4 大类别
- **localStorage 自动保存草稿**，防止误刷新丢失
- **字段级权限控制**：发起者填第1类、责任单位填2-3类、质量部门填第4类

### 模块2：4大类问题报告结构
- **第一类**：问题描述及现状（客户/产品/材料/编号/责任单位/时间/数量/类型/描述/图片）
- **第二类**：根本原因分析（鱼骨图 Ishikawa / 5Why 分析法，5M1E 分类标签）
- **第三类**：纠正预防措施（在制品处理 + 动态预防措施列表，含责任人/计划日期/实际日期/状态）
- **第四类**：效果验证（验证结论/方法/日期/验证人/报告上传/问题关闭）

### 模块3：数据看板与智能表格
- **统计卡片**：问题总数/整改中/已完成/已拖期
- **柱状图**：责任单位问题数量（按状态堆叠）
- **环形图**：整改情况占比
- **趋势折线图**：近30天问题发生趋势
- **拖期警示列表**：自动筛选拖期记录，红色高亮
- **数据表格**：列排序、筛选器（状态/单位/时间）、分页（50/100/200条）、全局搜索
- **状态标签色**：未开始(灰)/整改中(橙)/已完成(绿)/已关闭(蓝)/已拖期(红闪烁)

### 模块4：单问题透视（PPT四宫格）
- 点击列表行 → 全屏弹窗
- 2×2 四宫格布局：问题描述 | 根因分析 | 整改措施 | 效果验证
- 支持打印（适配A4纸）

### 模块5：腾讯文档接入与定时推送
- **腾讯文档同步**：已成功创建在线文档（https://docs.qq.com/doc/DQmdacXNyand2Vm9U）
- **质量周报生成**：自动汇总统计数据生成周报
- **定时推送配置**：频率/时间/内容/渠道/接收人，可创建自动化定时任务

## 技术架构

| 层级 | 技术 |
|------|------|
| 前端 | HTML5 + CSS3 + 原生 JavaScript（无框架依赖） |
| 图表 | Chart.js 4.4.1 |
| 字体 | Plus Jakarta Sans + JetBrains Mono |
| 数据存储 | localStorage（可扩展为后端 API） |
| 文档同步 | 腾讯文档 MCP |
| 定时任务 | WorkBuddy Automation |

## 设计系统
- **美学方向**：工业精密 × 数据清晰
- **色彩**：OKLCH 色彩空间，钢蓝主色，冷色调中性灰
- **间距**：4pt 基础系统
- **暗色模式**：完整支持，平滑切换
- **响应式**：桌面/平板/移动端适配
